# Content Repos

## Book Repo (Commentaries, Textbooks)

```
oc-dsgvo/
  meta.yaml              # Required: self-description
  toc.yaml               # Required: table of contents (SSOT for order)
  jura.csl               # Required: citation style
  references.yaml        # Required: bibliography
  divs.yaml              # Optional: custom div mapping
  Makefile               # make pdf
  pandoc.yaml            # Pandoc config
  pandoc/                # Templates, filters, fonts
  content/               # Markdown files
  .github/workflows/build-pdf.yml
```

### meta.yaml (required fields)

```yaml
slug: "oc-dsgvo"
type: "book"                    # book | journal
title: "OpenCommentary DSGVO"
title_short: "OC-DSGVO"
lang: "de"
license: "CC-BY-SA-4.0"
numbering: "commentary"
csl: "jura.csl"
bibliography: "references.yaml"
comments_on: "dsgvo"            # Optional: slug of the commented law
translations: ["en"]
editors:
  - name: "Max Mustermann"
    orcid: "0000-0000-0000-0000"
```

### toc.yaml

```yaml
contents:
  - file: preface.md
    title: Preface
  - file: art-5.md
    title: "Art. 5 – Principles"
    provisions: [5]
```

## Law Collection Repo

```
openlex-demo-law/
  sync.yaml              # Central config: laws, sources, metadata
  .github/workflows/sync.yml
  dsgvo/                 # Auto-generated by sync
  ai-act/
  bgb/
  stgb/
```

- `sync.yaml` contains all metadata per law (slug, source, title, title_short, unit_type, license, translations, supplements).
- Law files are pure Markdown without frontmatter — just the legal text.
- All metadata is derived: number from filename, title from `sync.yaml`, prefix from `unit_type`.
- Sync via [openlex-lawsync](https://github.com/openlex-project/openlex-lawsync) GitHub Action.
- Sources: GII (German federal law via Cloudflare proxy), EUR-Lex (Cellar/Formex).
- Supplements: recitals (`rec-{nr}.md`), annexes (`annex-{nr}.md`).
